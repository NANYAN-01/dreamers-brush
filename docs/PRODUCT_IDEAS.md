# 童梦画笔 (Dreamer's Brush) — 产品拓展想法与路线图

> 记录时间：2026-06-24
> 来源：基于现有架构的头脑风暴与用户讨论

---

## 当前产品现状

童梦画笔目前是一个基于 Web 的前后端分离项目：
- **前端**：React 18 + TypeScript + Webpack + TailwindCSS，运行在 `localhost:3266`
- **后端**：Express 5（纯 JS）+ MySQL，运行在 `localhost:3001`
- **AI 能力**：阿里云 DashScope 图像生成（通义千问/万相）
- **核心功能**：用户输入角色和动作描述，AI 生成对应风格的儿童绘本插图

---

## 想法一：微信小程序化

### 背景
将现有 Web 应用迁移或重写为微信小程序，利用微信生态触达更多家庭用户。

### 核心优势
- **触达更广**：微信生态天然适合儿童产品，家长无需下载 App
- **分享裂变**：一键分享到微信群、朋友圈，传播成本极低
- **家长付费意愿强**：儿童教育类产品在小程序生态变现路径清晰

### 技术方案

#### 方案 A：原生微信小程序（推荐）
- 前端用原生微信小程序重写
- 后端 Express API 直接复用，增加小程序登录接口（`wx.login` 获取 OpenID）
- 图片生成走后端转发（已满足，无跨域问题）

#### 方案 B：Taro / UniApp 跨端
- 一套代码同时支持小程序 + H5 + App
- 但针对童梦画笔这种重交互、重 UI 的产品，原生体验更好

### 小程序特有考虑
- **内容安全**：微信对 AI 生成内容有审核要求，需接入微信内容安全 API
- **图片存储**：小程序 `wx.downloadFile` 有域名限制，需配置 `downloadFile` 域名白名单
- **用户体系**：微信小程序用户无需注册，直接用 `openid` 作为用户 ID
- **订阅消息**：生成完成后推送给家长"故事已生成，快来查看"

## 想法二：有声故事本（用户重点提出）

### 核心概念
将静态的图片生成升级为**"翻页有声绘本"**：
- 每页故事配一段音频
- 翻页时自动播放该页音频
- 支持"自动播放整本故事"模式（翻一页播一页）

### 实现方案

#### 1. TTS（文本转语音）
```
用户输入的文字描述 → 后端调用 TTS API → 返回音频 URL → 前端播放
```

**推荐 TTS 服务对比**：

| 服务 | 优势 | 成本 |
|------|------|------|
| **阿里云语音合成** | 中文自然度高，儿童音色可选 | 按量付费 |
| **微软 Azure TTS** | 情感丰富，支持 SSML 精细控制 | 每月有免费额度 |
| **OpenAI TTS** | 超逼真，情感自然 | 价格稍贵 |

#### 2. 翻页动画 + 音频同步
- 每页故事配一段音频
- 翻页时自动播放该页音频
- 支持"自动播放整本故事"模式

#### 3. 音频缓存
- 首次生成后缓存音频文件，避免重复调用 TTS
- 存储到 `server/uploads/audio/` 或云存储

## 想法三：AI 语音克隆（更激进的想法）

### 核心概念
> **"用妈妈的声音讲故事"**

- 家长录音 5 分钟样本 → AI 克隆音色
- 生成的故事本用家长的声音朗读
- 情感价值拉满，家长付费意愿极强

### 技术方案
- 阿里云语音合成支持音色定制
- 或接入第三方语音克隆服务（如 MiniMax、ElevenLabs 等）

## 建议实施路线图

### 第一阶段：Web 版有声绘本（验证 PMF）
1. 增加"故事本"概念（多页编辑，每页文字 + 图片）
2. 集成 TTS（阿里云/微软），翻页自动朗读
3. 导出故事本为视频 / 长图 / 可分享链接
4. **验证产品 PMF（产品市场匹配）**

### 第二阶段：微信小程序迁移
- 小程序审核严格，建议 Web 版跑通后再迁移
- Web 版作为"深度编辑器"，小程序作为"轻量阅读/分享端"
- 数据打通，账号体系统一

### 第三阶段：高级功能
- AI 语音克隆
- 社交分享 / 社区画廊
- 订阅制 / 付费解锁
